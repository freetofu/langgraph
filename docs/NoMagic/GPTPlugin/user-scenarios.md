# CATIA Magic + GPT 연계 사용자 시나리오 초안

아래 시나리오는 `user-requirements.md`의 요구사항(UR-01~UR-10)을 기반으로 구성한 **사용자 중심 흐름**이다.

## 시나리오 1: 요구사항 도출 및 정제 (UR-01, UR-07)
- **배경**: 시스템 엔지니어가 신규 프로젝트 초기에 요구사항을 빠르게 정리해야 한다.
- **목표**: 이해관계자 인터뷰 내용을 바탕으로 요구사항 후보군을 도출하고 품질을 개선한다.
- **흐름**:
  1. 사용자가 GPT 패널에 인터뷰 메모를 붙여넣는다.
  2. GPT가 요구사항 후보군을 리스트로 제안한다.
  3. 사용자가 “검증 가능성” 관점에서 개선 요청을 한다.
  4. GPT가 불명확한 표현을 수정하고 측정 가능한 형태로 제시한다.
- **기대 결과**: 품질 기준을 충족하는 요구사항 초안이 생성된다.

## 시나리오 2: 모델 요소 초안 생성 (UR-02)
- **배경**: 아키텍트가 초기 모델 구조를 빠르게 잡아야 한다.
- **목표**: 요구사항을 기반으로 블록/액터/인터페이스 초안을 생성한다.
- **흐름**:
  1. 사용자가 핵심 요구사항과 시스템 범위를 GPT에 입력한다.
  2. GPT가 주요 블록과 액터 후보를 제안한다.
  3. GPT가 블록 간 관계(구성, 의존)를 요약한다.
  4. 사용자가 제안을 검토하고 모델에 반영할 요소를 선택한다.
- **기대 결과**: 초기 모델 구조 초안이 확보된다.

## 시나리오 3: 요구사항-모델 추적성 생성 (UR-03)
- **배경**: 요구사항과 모델 요소 간 추적성이 부족하다.
- **목표**: 추적 링크 후보를 생성하고 영향 분석을 수행한다.
- **흐름**:
  1. 사용자가 특정 요구사항 세트를 선택한다.
  2. GPT가 연결될 가능성이 높은 모델 요소를 추천한다.
  3. 변경된 요구사항에 대해 영향받는 요소 목록을 요약한다.
- **기대 결과**: 추적성 강화 및 변경 영향 분석 지원.

## 시나리오 4: 모델 품질 점검 (UR-04)
- **배경**: 모델 구조가 커지면서 오류/누락 위험이 증가한다.
- **목표**: 구조적 품질 문제를 사전에 점검한다.
- **흐름**:
  1. 사용자가 GPT에 “모델 점검”을 요청한다.
  2. GPT가 체크리스트(누락된 인터페이스, 미완성 관계 등)를 제공한다.
  3. 사용자가 검사 결과를 반영해 모델을 보완한다.
- **기대 결과**: 구조적 결함을 사전에 발견하여 수정한다.

## 시나리오 5: V&V 계획 초안 생성 (UR-05)
- **배경**: 검증 담당자가 요구사항 기반 테스트 계획을 수립해야 한다.
- **목표**: 요구사항과 연결된 검증 항목을 작성한다.
- **흐름**:
  1. 사용자가 GPT에 특정 요구사항을 입력한다.
  2. GPT가 시험/검증 항목과 절차를 제안한다.
  3. 사용자가 프로젝트 상황에 맞게 수정한다.
- **기대 결과**: V&V 계획 초안이 빠르게 확보된다.

## 시나리오 6: 규격 준수 점검 (UR-08)
- **배경**: ISO 26262/21434 등 규격 준수가 필수이다.
- **목표**: 규격 관점에서 누락 요소를 점검한다.
- **흐름**:
  1. 사용자가 적용할 규격을 GPT에 지정한다.
  2. GPT가 체크리스트와 누락 가능 항목을 제안한다.
  3. 사용자가 보완 계획을 수립한다.
- **기대 결과**: 규격 대응 체계가 강화된다.

## 시나리오 7: 작업 자동화 아이디어 제안 (UR-09)
- **배경**: 반복 작업(표 생성, 템플릿 적용)이 많다.
- **목표**: 자동화 후보를 식별한다.
- **흐름**:
  1. 사용자가 반복 작업 목록을 GPT에 입력한다.
  2. GPT가 Magic API/스크립트 기반 자동화 아이디어를 제시한다.
  3. 사용자가 우선순위를 정한다.
- **기대 결과**: 반복 작업 감소 및 생산성 향상.

## 시나리오 8: 세션 컨텍스트 유지 (UR-06)
- **배경**: 프로젝트 진행 중 대화 맥락을 유지해야 한다.
- **목표**: 이전 대화와 변경 이력을 참고한 응답을 받는다.
- **흐름**:
  1. 사용자가 이전 대화와 연속된 질문을 한다.
  2. GPT가 누적된 컨텍스트를 반영하여 답변한다.
- **기대 결과**: 연속적인 설계/검토 흐름 유지.

## 시나리오 9: 사용자 편의성 확보 (UR-10)
- **배경**: 사용자는 모델링 화면을 떠나지 않고 GPT를 사용하고 싶다.
- **목표**: 패널에서 즉시 질문하고 요약/상세 모드로 확인한다.
- **흐름**:
  1. 사용자가 패널을 열고 질문을 입력한다.
  2. GPT가 요약 답변을 먼저 제공한다.
  3. 사용자가 상세 모드로 확장해 내용을 확인한다.
- **기대 결과**: 작업 중단 없이 GPT 활용 가능.

## 시나리오 10: 통합 워크플로우 예시 (UR-01~UR-10)
- **배경**: 프로젝트 전 주기에 걸쳐 GPT를 활용한다.
- **목표**: 요구사항 → 아키텍처 → 검증까지 일관된 지원을 받는다.
- **흐름**:
  1. 요구사항 도출 및 정제.
  2. 모델 요소/관계 초안 생성.
  3. 추적성 연결과 영향 분석.
  4. 모델 품질 점검 및 규격 준수 검토.
  5. V&V 계획 수립.
- **기대 결과**: 전 생애주기에서 GPT 활용 체계 확립.
