# CATIA Magic + GPT 연계 사용자 요구사항 초안

## 1. 목적 및 범위
- CATIA Magic(기존 MagicDraw 포함)에서 모델링 업무를 수행하는 사용자가 GPT를 활용해 **요구사항 정의, 모델 설계, 검증 활동**을 효율화한다.
- 본 문서는 **사용자 관점의 요구사항 초안**으로, 기능 범위와 기대 효과를 정리한다.

## 2. 사용자 페르소나
- **시스템 엔지니어**: 요구사항 수집/정제, 모델 구조 설계, 검증 시나리오 정리에 GPT를 활용.
- **아키텍트**: 상위 구조/인터페이스 정의, 대안 비교 분석 지원 필요.
- **검증 담당자**: 모델 기반 V&V 계획 수립 및 체크리스트 자동화에 관심.

## 3. 사용자 요구사항(안)

### UR-01: 요구사항 도출 지원
- 사용자는 자연어 입력으로 **요구사항 후보군**을 생성할 수 있어야 한다.
- GPT는 **요구사항 품질 기준(명확성, 검증 가능성, 추적성)**에 맞춰 정제 제안을 제공해야 한다.

### UR-02: 모델 요소 추천 및 초안 생성
- 사용자는 GPT에게 **블록, 액터, 인터페이스, 파트 등 모델 요소 초안** 생성을 요청할 수 있어야 한다.
- GPT는 입력된 요구사항과 시스템 범위를 바탕으로 **관계(연관/구성/의존)** 후보를 제안해야 한다.

### UR-03: 요구사항-모델 추적성 지원
- 사용자는 GPT를 통해 요구사항과 모델 요소 간 **추적 링크 후보**를 자동 생성할 수 있어야 한다.
- GPT는 변경된 요구사항에 대해 **영향받는 모델 요소 목록**을 요약해 제공해야 한다.

### UR-04: 모델 검토 및 품질 점검
- 사용자는 모델의 구조적 이슈(불완전한 관계, 누락된 인터페이스 등) 점검을 GPT에 요청할 수 있어야 한다.
- GPT는 **검토 체크리스트**와 **잠재 리스크**를 제안해야 한다.

### UR-05: V&V(Verification & Validation) 지원
- 사용자는 특정 요구사항을 검증하기 위한 **시험/검증 항목 초안**을 GPT로부터 받을 수 있어야 한다.
- GPT는 요구사항과 모델의 연결을 고려해 **검증 시나리오**를 요약해야 한다.

### UR-06: 사용자 입력 이력 기반 컨텍스트 유지
- 사용자는 세션 내에서 **이전 대화와 모델 변경 이력**을 컨텍스트로 이어서 활용할 수 있어야 한다.
- GPT는 동일 프로젝트 내에서 **누적된 대화 맥락**을 고려하여 답변해야 한다.

### UR-07: 시스템 엔지니어링 단계별 가이드
- 사용자는 GPT에게 **현재 단계(요구사항, 아키텍처, 상세설계 등)**에 맞는 가이드를 요청할 수 있어야 한다.
- GPT는 단계별 산출물 예시와 체크리스트를 제안해야 한다.

### UR-08: 안전/보안/규격 준수 지원
- 사용자는 ISO 26262, ISO 21434 등 규격 기반 **준수 항목 체크**를 GPT에 요청할 수 있어야 한다.
- GPT는 해당 규격 관점에서 **누락 항목/리스크**를 요약해야 한다.

### UR-09: 모델링 작업 자동화 제안
- 사용자는 반복적인 모델링 작업(템플릿 생성, 표 생성 등)을 GPT로 자동화 제안받을 수 있어야 한다.
- GPT는 Magic API/스크립트 활용 가능성을 기반으로 **자동화 절차 초안**을 제시해야 한다.

### UR-10: 사용 편의성 및 접근성
- 사용자는 모델링 화면을 벗어나지 않고 **패널 형태로 GPT를 즉시 호출**할 수 있어야 한다.
- GPT 응답은 **요약/상세** 모드로 제공되어야 한다.

## 4. 비기능 요구사항(안)
- **성능**: 입력 후 응답까지 평균 3~5초 이내를 목표로 한다.
- **보안**: 프로젝트 데이터와 대화 내용은 **외부 유출 방지 정책**을 준수해야 한다.
- **감사/추적**: GPT의 제안 및 사용 기록을 **감사 로그**로 남길 수 있어야 한다.
- **확장성**: 다양한 SE 표준(예: INCOSE, UAF, SysML v1/v2)을 적용 가능한 구조여야 한다.

## 5. 향후 상세화 항목
- 모델 요소별 생성 규칙(네이밍 규칙, 스테레오타입 매핑)
- 프로젝트별 템플릿/도메인 지식 사전 연동 방식
- 규격별 검증 체크리스트 템플릿
